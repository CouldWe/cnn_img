# 1 设计背景

本设计所面向的场景为高实时性应用场景，需要达到每秒 600K 个输入特征图的处理能力。在该场景下，重点评价指标为芯片面效， $\mathrm { F o M } =$ Speed/Area (单位: 1/mm2)，因此主要设计目标为提升实时处理能力，同时降低电路的面积开销。为了达到以上设计目标，采用全展开的 PE 阵列进行卷积计算，来最大化电路的实时处理能力。

# 2.1模块互联

电路的模块连接关系如图 2 所示，主要模块包括三层脉动阵列进行卷积运算（SA1、SA2、SA3）、三级缓存（Buffer1、Buffer2、Buffer3）、最大池化层（Maxpool）和全连接层（FC）。数据流从输入开始，依次经过SA1进行第一层卷积，偏置、重量化并激活后结果存入Buffer1；随后从 Buffer1 读出并送入 SA2 进行第二层卷积，同理，结果存入 Buffer2；Buffer2 的输出进入 SA3 完成第三层卷积，先输出的一列结果存入 Buffer3，后输出的结果和缓存层中的结果一起进行最大池化；池化结果进入 FC 完成全连接运算，最后经过 Sigmoid 输出层得到注意力分数。

# 2.2 时序结构与硬件间详细连接关系

在顶层模块，引入了一个计数器 cnt，在顶层模块 enable 信号拉高时，每个时钟周期加一。令 $\mathrm { c n t } = 0$ 为 T0 时刻， $\mathrm { c n t } = 1$ 为 T1 时刻，以此类推。cnt 作为顶层控制信号，决定了各个模块的开关与其它控制信号的升降。

T0-T119时，输入特征图脉动进入 SA1，进行第一层卷积运算。SA1 的输出值将进行偏置运算，运算后的信号输入一级 flip-flop（FF），FF 输出信号输入级联的重量化/激活单元，最终输出值接入第一层缓存层。

在 T19-T38/T49-T68/T79-T98/T109-T128 时，第一缓存层将接收到 SA1 输出的有效卷积结果，其使能信号将提前一个周期拉高。在 T90-T129，其将接收到读地址信号，并在下一个周期输出对应地址的三个相关数据。其输出将作为 SA2 的输入。

SA2 的输出同 SA1 一样，将依次经过偏置、重量化和激活，其最终输出结果直接与一级 FF 相连，该 FF 的输出将作为 SA3 的输入信号，SA3 的输出经过偏置、重量化和激活后将分别输入进第三层缓存层和最大池化层。其中在 T107-T124，第三层缓存层将接收 SA3的输出值；在 T127-T144，第三层缓存层中的数据将被依次读出，与此同时，最大池化层将同时接收 SA3 的输出值和从第三层缓存层读出的数据，进行最大池化。最大池化层的输出直接与全连接层输入相连。

在 T128-T145，全连接层被使能，开始接收最大池化结果。全连接层的输出将偏置、重量化后输入到 Sigmoid 层，Sigmoid 的输出即为整个顶层模块的输出。

在 T153 与 T154，顶层模块的输出使能信号拉高，其将在两个周期依次输出两个注意分数值。

因此，本电路能够在 155个时钟周期完成一张输入特征图的处理。经过综合仿真，该电路的最大工作频率为 97MHz，其特征图最大处理能力为 625.8K/s。

# 3 电路架构

# 3.1 数据预处理

在设计规范中，输入特征图的尺寸为(1, 30,10)，而本设计采用了权重固定的 PE 阵列进行常规卷积运算，输入特征图的数据按行脉动进入 PE 阵列，其相当于 PE 阵列窗口逐行扫描输入特征图。当 PE 阵列完成整 7列的数据扫描之后，将右移 1 列重新进行数据扫描，直到扫描完所有列的数据。对于第一卷积层的单个通道，其宽度为 7，则每次脉动进入 PE 阵列的数据数目应当为 7，因此需要对输入特征图进行数据预处理，使之与 PE 阵列的输入宽度匹配，处理后的输入特征图尺寸变为(1, 120, 7)。

# 3.2 第一卷积层

第一个常规卷积层输入数据尺寸为(1, 120, 7)，输入数据逐行脉动进入 PE 阵列之中，120个周期可输入完全部输入数据。常规卷积层采用全展开的架构，因此其 32个通道将同时并行进行数据处理，每个周期将输出 32个通道的卷积计算结果。另外对于单个卷积通道，其输出结果为 77 个 PE 单元运算结果的累加，为减小关键路径，我们采用流水线加法树进行累加运算。

PE 单元采用权重固定，数据流动的架构。我们将每个 PE 内部的权重直接写入其中，每一个时钟周期 PE 单元将会接收上一个 PE 单元脉动输入的数据，并将当前数据脉动输出给下一个 PE 单元，同时会输出当前权重和数据的乘积。

# 3.3 第一缓存层

第一缓存层基于 SRAM 实现，能够根据整个迭代周期（将输入特征图第一行数据输入常规卷积层到输出层输出最终结果的时间称为一个迭代周期）的时序存入常规卷积层的有效输出数据。由于常规卷积层每次同时输出 32个 8 位数据，因此将缓存层的输入位宽设置为256位，缓冲层每一个地址都存储了 32个通道对应的中间数据，由于中间数据一共 $2 0 { \times } 4$ 个，则第一缓冲层的尺寸为(1,20,4)（在实际实现时，最后一列数据并不需要全部存储，可使用一条较短的移位寄存器链进行数据存储）。由于在深度可分离卷积层中，单通道卷积核宽度，即输入数据个数为 3，因此第一缓冲层每次需要输出 3 个地址的数据，其位宽需要设置为 768位（ $3 { \times } 3 2 { \times } 8 { \ }$ ）。由于我们能够使用的 SRAM 字宽最大为 128位，因此需要使用两个 SRAM拓展成位宽为 256的 SRAM 以存储单个地址的数据；同时由于 SRAM 无法同时读写数据，因此需要使用多个 SRAM 进行数据读写，考虑到涉及同时写入一个数据并读出三个数据的操作，我们最少需要使用 4 个 SRAM；最后注意到整个缓冲层的数据按列输入，其前三列数据需要进行长时存储或数据复用，而最后一列数据输入到输出的时间相隔较短且不需要进行数据复用，因此最后一列数据可以使用一个较短的移位寄存器进行数据存储，这能有效简化 SRAM 的输入输出时序。

第一层缓存层的输入输出端口如图 10 所示。当 enable信号拉高时，可以通过 256 位的输入端口 data_in（输入 32个 8bit 数据）对缓存层写入数据，同时开启该缓存层的时序迭代；其地址输入信号为 raddr，其范围为 0-39（20行，卷积核按列扫描 2轮），在指定的时序周期，缓存层将根据 raddr的值，在下个时钟周期输出 3 个 256位的输出值。

# 3.4 第二卷积层

第二个深度可分离卷积层的实现方式类似于常规卷积层，都采用了 32通道并行的全展开架构，其乘累加也都是通过流水加法树实现的，在此不再详述。

# 3.5 第二缓存层

由于第三层逐点卷积层每个通道的输入数据为第二层卷积运算结果的单个数据点，因此第二层卷积结果可以直接输入第三层卷积层，但为了减小关键路径，仍然选择在两层卷积层中加入一层缓冲层，该缓冲层相当于一个输入输出位宽为 32 的 flip-flop。

# 3.6 第三卷积层

对于第三层逐点卷积层，其本质就是每通道中 32 个数据的乘累加线性运算，不存在数据复用，因此该层我们可以直接用流水加法树进行构建，其顶层顶层电路与内部电路如图12与图 13所示。

# 3.7 第三缓存层

第三层卷积的结果将用以最大池化，考虑到第三层卷积结果尺寸为 $1 8 { \times } 2$ （视 32个通道的数据为同一数据点），而其输出顺序为按列输出，因此可以先存储第一列中间结果，将第二列卷积结果与缓冲层中存储的第一列结果输入到最大池化层进行池化操作。

因此，第三层卷积层只需要存储逐点卷积层输出的第一列卷积结果。我们依旧采用 sram进行存储，类似于第一层缓存层，第三层缓存层的 sram 依旧需要两个 128位的 sram 拼接成一个 256 位的 sram。

# 3.8 最大池化层

最大池化层采用时序逻辑实现，其输入数据为第三层卷积结果和第三层缓冲层的输出结果，在相邻两个周期输入共 4个数据，在接下来两个周期，最大池化层将输出以上 4 个数据的最大值，接收新的待池化的 4 个数据。因此最大池化层将每两个周期输出一次池化结果，每个池化结果保持两个周期。

# 3.9 展平/全连接层

展平操作即为池化结果数据标号的线性变换，不存在任何逻辑运算。全连接运算本质为两个通道的 288个数据的乘累加运算，由于该乘累加规模巨大，同时考虑到展平层的数据是逐步填充的，具体为每 2 个时钟周期填充等距的 32个数据，因此可以采用折叠架构进行全连接运算。采用 1:18 的折叠方式，共设置了 32 个折叠单元。我们将展平层分为 32 组，每组 9 个数据，每一组数据对应一个折叠单元。每个数据保持 2 个周期，则在第一个周期，输入数据与第一个通道的对应权重相乘，并更新部分和；在第二个周期，输入数据与第二个通道的对应权重相乘，并更新部分和。18 个时钟周期结束后，每个 PE 单元都存储了两个通道的 9 个数据的乘累加结果。最后再通过流水加法树分别计算这个两个的通道的 32个数据的

累加和，得到最后的输出结果。

# 3.10 输出层

采用查找表实现输出层 Sigmoid，输出层的输入为 8 位数据，因此根据映射关系构建了一个 256规模的查找表。输入到输出为组合逻辑。

# 3.11 重量化/激活层

在卷积层输出及全连接层输出后，需要对输出数据进行重量化与 Relu 激活，由于这两层相隔较近，逻辑简单，且不同激活层与重量化层的实现大同小异，因此在此处一并介绍。

对于重量化层，首先需要将输入数据乘以一个常量，然后再进行二的 n 次幂的除法操作得到最终的量化结果。由于乘法位宽较大，因此将该处的乘法转换为移位加操作。为了减小关键路径，将移位加分为两步：在第一个时钟周期，分别得到两个移位加的部分和；在第二个时钟周期，将这两个部分和相加得到最终的乘积。将二的次幂的除法操作转换为算数右移位，最后再根据右移后的结果是否溢出决定重量化的最终输出。（对于逐点卷积层输出结果的重量化，其移位加的被加数相对较少，不需要分为两部分进行相加，但为了减少总的组合路径长度，依旧引入了一级寄存器。）量化模块内部电路为图 19。

激活层采用简单组合逻辑实现，其将输入数据与 0 进行比较，输出两者中的最大值。

# 4 电路性能分析

经过逻辑仿真验证，本电路能够在 155个时钟周期处理完一张输入特征图，在 100M的工作频率下能够达到 645K的处理速度，能够满足高实时性的应用场景需求。

本架构的运算密集层主要为 SA1，需要 120个时钟周期才可完成一张输入特征图的运算，其余卷积层如 SA2，只需要 40个时钟周期即可完成一张输入特征图的处理，因此 SA1 限制了本架构的理论最小处理周期。

由于本架构采用统一时序控制信号，使得电路在当前特征图处理完成之前无法开始处理下一张图，电路的并行计算性能并没有完全发挥；同时 SA1 按列单向扫描输入特征图，数据复用未实现最大化。在未来的性能优化中，可以考虑更改顶层时序控制逻辑，通过握手协议完成模块间控制信号的传递，提高电路的并行性，降低迭代周期长度；同时还可以实现SA1 行列双向扫描，将架构的理论小处理周期降至 90 个时钟周期，进一步提高电路整体速度。
